# 🚀 Development Rules

## 1. Plan First, Code Later

- Always create a clear plan before writing code.
- If something goes wrong mid-way:
  - Stop immediately.
  - Re-evaluate and update the plan.
- Never force a broken approach to continue.

---

## 2. Delegate Smartly

- Assign complex tasks to sub-agents when possible.
- Keep the main context clean and focused.
- Prefer allocating more compute/resources over handling everything manually.

---

## 3. Continuous Self-Improvement Loop

- After each task:
  - Extract lessons learned.
- In the next session:
  - Review previous lessons.
  - Apply improvements proactively.
- Goal: Reduce repeated mistakes over time.

---

## 4. Proof of Completion

- A task is NOT complete until:
  - Tests are executed.
  - Logs are checked.
- No assumptions. Only verified results count.

---

## 5. Autonomous Debugging

- When encountering bugs:
  - Investigate logs to find the root cause.
  - Fix the issue immediately.
- Do not rely on external guidance.
- Aim for full ownership of problems and solutions.

---

# 🧠 Mindset

- Think systematically.
- Act decisively.
- Improve continuously.
- Verify everything.
